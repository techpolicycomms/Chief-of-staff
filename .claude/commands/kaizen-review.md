Run the Sunday kaizen review:

1. Read the most recent `memory/kaizen-research-*.md`.
2. Read the Frictions sections of the last 7 daily notes.
3. Read `MEMORY.md` and `scripts/` filenames to ground proposals in
   what actually exists.
4. Walk me through:
   - Top 3 external findings worth borrowing
   - This week's recurring frictions
   - 1-3 concrete proposed changes to the system itself
5. For each proposal, note: what we'd add, what we'd cut, and what
   we'd lose. Bias toward removal over addition.
6. Wait for my decision on each before making any code or memory edits.
