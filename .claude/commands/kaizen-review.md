Run the Sunday kaizen review:

1. Read the most recent `memory/kaizen-research-*.md`.
2. Read the Frictions sections of the last 7 daily notes.
3. Read `MEMORY.md`, `config/context-itu.md`, and `scripts/`
   filenames to ground proposals in what actually exists and in the
   ITU operating context.
4. Walk me through:
   - Top 3 external findings worth borrowing (favour things tested
     inside other UN agencies, treaty bodies, or large multilateral
     secretariats — generic startup hacks are usually a poor fit).
   - This week's recurring frictions, tagged by where they hit:
     CC prep, Council/CWG, Sector coordination, bilateral meeting
     prep, speech/drafting clearance, travel logistics, email triage.
   - 1-3 concrete proposed changes to the system itself.
5. For each proposal note: what we'd add, what we'd cut, what we'd
   lose. Bias toward removal over addition. Flag anything that would
   reduce the SG's clearance loop or speed up draft turnaround.
6. Wait for my decision on each before making any code or memory edits.
