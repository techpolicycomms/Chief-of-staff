# Daily note template

You are writing today's daily note at `memory/YYYY-MM-DD.md`. This is a
raw log, not a report. Short entries, factual, with timestamps when the
source has them. Append throughout the day.

## Sections

```
# YYYY-MM-DD

## Meetings
- HH:MM <attendee> — <one-line outcome>. Action items: [...]

## Email triage
- <sender> re: <subject> — <decision: action/archive/reply drafted>

## Tasks
- Added: ...
- Completed: ...
- Slipped: ...

## Decisions
- <short decision + rationale>

## Threads / context
- <one-liners worth remembering later>

## Frictions (for kaizen)
- <anything that felt like drag — a bad filter, a missing brief, a nag>
```

## Rules

- If nothing happened in a section, omit it. Empty sections are noise.
- Write in past tense, third person or neutral. Not "I did X" — "<user>
  did X" or just "X".
- Never fabricate. If you don't have a fact, leave it out.
- Keep each bullet under one line when possible.
- Pull names from `config/people.yml` so slugs match `people/<slug>.md`.
